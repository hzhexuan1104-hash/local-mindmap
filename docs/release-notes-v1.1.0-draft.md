# v1.1.0 发布说明草稿

发布日期：待定

## v1.1 目标

v1.1 第一批目标是在不破坏 v1.0.0 Web 版能力的前提下，补齐桌面端最小运行基础，并增强基础编辑体验。

本版本继续坚持：

- 纯本地运行。
- 不上传用户文件或节点数据。
- 不引入云端同步、用户登录或协同编辑。
- 不执行第三方插件 JavaScript。
- `.lmind` 继续保持标准 JSON。

## 新增功能

- Tauri v2 桌面端最小壳：
  - 新增 `src-tauri/` 配置。
  - 新增 `npm run tauri:dev`。
  - 新增 `npm run tauri:build`。
  - 桌面窗口加载当前 Vite + React 应用。
- 节点拖拽自由布局基础版：
  - 节点可鼠标拖拽。
  - 拖拽时节点和连线实时更新。
  - 拖拽结束后保存 `position?: { x: number; y: number }`。
  - 保存和打开 `.lmind` 后保留位置。
- 重新自动布局：
  - 顶部视图区域和画布右键菜单提供“重新自动布局”。
  - 清除节点 `position` 并恢复左到右自动布局。
  - 支持撤销 / 重做。
- 多选节点基础版：
  - Ctrl + 点击、Shift + 点击追加选择。
  - 支持批量删除。
  - 支持批量切换节点类型。
  - root 节点不允许被批量删除。
- 右键菜单基础版：
  - 节点右键菜单支持新增、编辑、删除、复制文本、折叠/展开、切换节点类型和重新自动布局。
  - 画布右键菜单支持新建、一键居中、重新自动布局、展开全部、折叠全部、导出 PNG/JPG。
- UI 小幅优化：
  - 顶部按钮按文件、导入导出、工具分组。
  - 工具区域支持换行。
  - 画布、多选、拖拽和右键菜单样式优化。
  - 备注面板滚动体验优化。

## 运行方式

Web 开发：

```bash
npm run dev
```

Web 构建：

```bash
npm run build
```

自动化测试：

```bash
npm run test
```

Tauri 桌面开发：

```bash
npm run tauri:dev
```

Tauri 桌面构建：

```bash
npm run tauri:build
```

Tauri 运行需要安装 Rust / Cargo 和目标平台系统依赖。

## 数据兼容

- v1.0.0 `.lmind` 文件没有 `position` 字段时仍可正常打开。
- 无 `position` 的节点继续使用左到右自动布局。
- Markdown / Excel 导入不会生成 `position`，仍使用自动布局。
- JSON 导入导出和 `.lmind` 保存会保留已有 `position`。
- 插件、主题、模板、自定义节点类型不受影响。

## 已知限制

- 当前环境缺少 Rust / Cargo，尚未在本机完成 Tauri 真实构建。
- Tauri 桌面端尚未生成正式安装包。
- 尚未验证 macOS、Linux、统信 UOS、银河麒麟和国产 CPU 架构。
- 节点拖拽仅保存自由位置，不调整父子层级和兄弟顺序。
- 多选基础版支持 Ctrl/Shift 点击，不支持框选。
- 右键菜单为基础版，暂不支持用户自定义菜单项。
- 仍未实现云同步、登录、协同编辑、插件市场、复杂智能排版、左右双向布局、鱼骨图。

## 手动测试清单

- 执行 `npm run test`。
- 执行 `npm run build`。
- 打开 Web 版，验证新建、保存、打开 `.lmind`。
- 拖拽节点，确认节点和连线实时更新。
- 保存拖拽后的 `.lmind`，重新打开后确认位置保留。
- 打开 v1.0.0 无 `position` 文件，确认自动布局正常。
- 点击“重新自动布局”，确认节点恢复左到右布局。
- Ctrl/Shift 点击多选节点，确认多选样式和备注面板主选中节点。
- 多选后批量删除，确认 root 不被删除且撤销/重做可用。
- 多选后批量切换节点类型。
- 右键节点，逐项验证节点菜单。
- 右键画布空白处，逐项验证画布菜单。
- 验证 Markdown / Excel / JSON 导入导出。
- 验证 PNG / JPG / TXT 导出。
- 验证模板、官方模板、主题、插件管理、查找替换和性能测试工具。
- 安装 Rust / Cargo 后执行 `npm run tauri:dev` 和 `npm run tauri:build`。
