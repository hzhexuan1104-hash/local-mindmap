# v1.1.1 发布说明草稿

发布日期：待定

## 版本定位

v1.1.1 是 v1.1 小版本 UI 信息架构优化版本，目标是在不破坏 v1.1.0 已有功能的前提下，让画布成为页面主区域。

本版本继续坚持：

- 纯本地运行。
- 不上传用户文件或节点数据。
- 不引入云端同步、用户登录或协同编辑。
- 不执行第三方插件 JavaScript。
- `.lmind` 数据结构不变。

## 主要变化

- 顶部导航压缩：
  - 顶部常驻高频入口：新建思维导图、保存 `.lmind`、打开 `.lmind`。
  - 其他操作进入文件、导入导出、编辑、视图、工具分组菜单。
- 非高频功能改为抽屉 / 面板：
  - 模板库。
  - 节点类型。
  - 查找替换。
  - 性能测试。
  - 插件管理入口。
  - 快捷键帮助。
- 主画布区域扩大：
  - 默认页面不再长期展开性能测试、查找替换、模板和节点类型大表单。
  - 画布在页面中占据主要空间。
- 画布工具悬浮化：
  - 放大、缩小、一键居中、重新自动布局和主题选择移动到画布内部悬浮工具条。
- 右侧备注面板可折叠：
  - 折叠后画布自动变宽。
  - 展开后继续显示当前节点 Markdown 备注、实时预览和放大预览。
- 新增专注模式：
  - 隐藏顶部工具区、左侧工具栏和右侧备注面板。
  - 画布最大化。
  - 保留画布缩放、居中、重新自动布局和退出专注模式入口。

## 兼容性

- 不修改 `.lmind` 文件结构。
- 不新增需要保存的 UI 状态。
- 备注面板折叠状态和专注模式仅为本地 UI 状态，不写入文件。
- v1.1.0 的节点拖拽、多选、右键菜单、导入导出、模板、插件、主题和性能测试逻辑继续复用原实现。

## 已知限制

- 本次不新增大型业务功能，不进入 v1.2。
- 顶部菜单和左侧抽屉属于 UI 入口调整，仍需要完整手动回归。
- 自动化测试仍以纯函数和数据处理逻辑为主，UI 布局依赖手动验收。
- 主 chunk 体积仍可能出现 Vite 体积提醒，不作为构建阻断错误。
- Tauri 真实安装包和跨平台桌面端验证仍属于后续任务。

## 测试建议

- 执行 `npm run build`。
- 执行 `npm run test`。
- 打开页面，验证画布区域明显变大。
- 验证顶部高频按钮和分组菜单。
- 验证左侧工具栏抽屉中的模板库、节点类型、查找替换、性能测试、插件管理和快捷键。
- 验证画布悬浮工具条的缩放、一键居中、重新自动布局和主题切换。
- 验证右侧备注面板折叠 / 展开、Markdown 编辑、实时预览和放大预览。
- 验证专注模式进入 / 退出。
- 回归新建、保存、打开、导入导出、拖拽、多选、右键菜单、模板、插件、节点类型和 GitHub Pages 构建。
