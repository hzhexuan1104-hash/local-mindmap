# 共享包说明

v1.3.0 新增两类独立共享包：节点类型包和模板包。共享包用于分享本地配置，不等同于 `.lmind` 导图文件，也不包含当前完整导图内容。

## 节点类型包

节点类型包用于分享自定义节点类型配置，例如图标、形状、背景色、边框色、文字颜色、字号、默认文本和默认备注。

格式示例：

```json
{
  "version": "1.0",
  "kind": "local-mindmap-node-type-pack",
  "meta": {
    "name": "节点类型包名称",
    "description": "节点类型包说明",
    "createdAt": "ISO 时间",
    "source": "local-mindmap"
  },
  "nodeTypes": []
}
```

节点类型包和 `.lmind` 的区别：

- 节点类型包只保存 `nodeTypes` 配置。
- 节点类型包不包含 `rootNode`。
- 节点类型包不保存当前导图的节点树、备注或主题状态。
- `.lmind` 仍是完整导图文件，可以携带当前导图使用的 `nodeTypes` 和 `rootNode`。

## 模板包

模板包用于分享本地自定义模板库。模板可包含模板自身的根节点结构、节点类型信息、主题和说明字段，以便导入后从模板库新建导图。

格式示例：

```json
{
  "version": "1.0",
  "kind": "local-mindmap-template-pack",
  "meta": {
    "name": "模板包名称",
    "description": "模板包说明",
    "createdAt": "ISO 时间",
    "source": "local-mindmap"
  },
  "templates": []
}
```

模板包和 `.lmind` 的区别：

- 模板包保存的是模板库配置，不代表当前打开的完整导图文件。
- 模板包不用于覆盖当前 `rootNode`。
- 模板包导入后写入当前浏览器 localStorage 中的本地模板库。
- `.lmind` 仍用于保存和打开完整导图。

## 如何导出

- 打开“节点类型”面板，点击“导出节点类型包”，生成 `local-mindmap-node-types.json`。
- 打开“模板库”面板，点击“导出模板包”，生成 `local-mindmap-templates.json`。
- 如果当前没有可导出的自定义节点类型或自定义模板，会显示空状态提示，不下载文件。

## 如何导入

- 打开“节点类型”面板，点击“导入节点类型包”，选择节点类型包 JSON。
- 打开“模板库”面板，点击“导入模板包”，选择模板包 JSON。
- 导入成功后会显示成功导入、跳过重复、重命名冲突和无效条目数量。
- 导入后的节点类型和模板会持久化到当前浏览器 localStorage，并可立即使用。

## 如何分享给其他人

将导出的 JSON 文件发送给其他人即可。对方在自己的浏览器中导入后，会写入对方本地的节点类型库或模板库。

共享包只在当前浏览器本地保存；如果换浏览器、换设备或清空站点数据，需要重新导入。

## 冲突处理

- ID 相同且内容相同：跳过，计入重复数量。
- ID 相同但内容不同：自动生成 `原ID-imported-N`，计入重命名冲突数量。
- name 相同但 ID 不同：允许导入，并在导入结果中提示同名数量。
- 缺少必要字段的节点类型或模板：跳过，计入无效条目数量。
- 空包：提示未找到可导入的节点类型或模板，不修改本地库。
- 非法 JSON：提示文件不是有效 JSON。
- kind 不匹配：提示这不是有效的节点类型包或模板包。

## 安全说明

- 共享包只是 JSON 配置。
- 导入共享包不会执行 JavaScript。
- 导入共享包不会上传文件。
- 导入共享包不依赖云端。
- 导入共享包不会修改 `.lmind` 基础结构。
- 导入节点类型包或模板包不会覆盖当前打开的导图结构。
- 当前仍是完全本地运行工具，不包含用户账号、团队权限、远程市场或协同编辑。

## 已知限制

- 当前没有远程市场。
- 当前没有团队权限。
- 当前没有自动同步。
- 导入后仅保存到当前浏览器 localStorage。
- 模板包和节点类型包是两类独立共享能力；模板依赖的节点类型信息仅在模板已有字段中保留。
