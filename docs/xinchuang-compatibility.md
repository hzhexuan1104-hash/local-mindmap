# 信创系统适配说明

## 1. 目标系统

- 统信 UOS 桌面版。
- 银河麒麟桌面版。
- Windows 10 及以上版本。
- macOS 11 及以上版本。

## 2. 目标架构

- x86 / x64。
- ARM64。
- 飞腾。
- 鲲鹏。
- 龙芯。

## 3. 当前 Web 版本适配情况

当前项目是 React + TypeScript + Vite 构建的前端 Web 应用，核心能力运行在浏览器中。

已具备的适配基础：

- 不依赖云端服务。
- 不需要用户登录。
- 数据通过本地文件导入导出。
- `.lmind` 文件是标准 JSON，跨系统可读。
- Markdown、Excel、JSON、图片导出和插件注册表均在本地处理。
- GitHub Pages 在线部署不影响本地运行，离线环境可通过本地构建产物或后续桌面端运行。

建议优先验证浏览器：

- Chrome。
- Edge。
- 统信 UOS / 银河麒麟内置或推荐浏览器。

## 4. 当前 Tauri 桌面端准备情况

v1.1 第一批已新增 Tauri v2 最小桌面壳：

- 桌面端加载当前 Vite 应用。
- Web 版和 GitHub Pages 构建方式保持不变。
- `.lmind` 数据结构保持标准 JSON。
- 浏览器版保存、打开、导入、导出逻辑不变。
- 未新增云端同步、用户登录、远程脚本或第三方插件 JS 执行能力。

当前环境尚未完成 Tauri 构建验证，原因是缺少 Rust / Cargo。后续在信创系统上验证前，需要先确认：

- Rust 工具链是否支持目标系统和目标 CPU 架构。
- Tauri 所需系统 WebView 是否可用。
- 打包产物是否能在统信 UOS、银河麒麟和国产 CPU 架构上启动。

## 5. 后续桌面端适配注意事项

- 桌面端应保留 Web 版功能，不改变 `.lmind` 数据结构。
- 打包工具链需分别验证 Windows、macOS、Linux / 信创系统。
- 如选择 Tauri，需要验证系统 WebView、Rust 工具链和目标 CPU 架构支持。
- 如选择 Electron，需要验证对应 CPU 架构的 Electron 运行时可用性和包体积接受度。
- 桌面端文件对话框应符合各系统交互习惯。
- 应避免引入必须联网的安装、启动或授权流程。

## 6. 文件系统注意事项

- 路径分隔符在 Windows 与 Linux/macOS 不同，代码中应避免硬编码路径。
- 文件名需要支持中文。
- `.lmind`、`.md`、`.xlsx`、`.json`、`.png`、`.jpg`、`.txt` 等后缀应在各平台可保存和打开。
- 桌面端后续如支持“最近文件”，只能保存本地路径，不上传。
- 文件读写应以用户主动选择为前提。

## 7. 字体和中文显示注意事项

- 中文字体在不同系统中默认字体不同，需要避免依赖单一字体。
- 建议 CSS 使用系统字体栈，优先保证中文可读性。
- 节点长中文、长英文、长数字和连续无空格文本必须自动换行。
- Markdown 预览、备注面板、插件面板、Excel 映射弹窗都需要验证中文显示。

## 8. 高 DPI / 2K / 4K 显示注意事项

- 在 125%、150%、200% 缩放比例下验证界面。
- 验证画布网格、节点边框、连线、按钮、弹窗是否清晰。
- 验证 PNG/JPG 图片导出在高 DPI 下是否清晰。
- 验证右侧备注面板、模板列表、插件管理面板不会横向溢出。

## 9. 离线运行要求

- 核心功能在断网环境下可用。
- 保存、打开、导入、导出都必须走本地能力。
- 插件基础版只从本地 JSON 安装，不从云端下载。
- localStorage 中的模板和插件注册表应在离线环境下正常读取。

## 10. 本地文件读写要求

- Web 版使用浏览器文件选择与下载能力。
- 桌面版后续可接入原生文件对话框。
- `.lmind` 必须保持标准 JSON。
- 打开错误文件时应给友好提示，不能崩溃。
- 保存文件时不得上传到云端。

## 11. 数据安全说明

- 不使用云端同步。
- 不上传用户文件。
- 不上传节点文本、备注、模板、插件配置或操作数据。
- 插件基础版不执行第三方 JavaScript。
- 后续桌面端如加入原生能力，应默认关闭不必要的网络和系统权限。

## 12. 风险清单

| 风险 | 说明 | 建议 |
|---|---|---|
| 不同系统文件选择器差异 | Web 和桌面端文件选择行为不同。 | 分平台验收打开、保存、导入、导出。 |
| 国产 CPU 架构依赖兼容 | Electron/Tauri 运行时和依赖包可能不支持所有架构。 | 提前做最小样例打包验证。 |
| 桌面端打包工具链兼容 | 信创系统上的构建链可能缺依赖。 | 优先在目标系统建立构建环境清单。 |
| Rust / Cargo 工具链缺失 | 当前环境无法运行 Tauri 构建。 | 安装 Rust / Cargo 后重跑 `npm run tauri:dev` 和 `npm run tauri:build`。 |
| Tauri WebView 依赖差异 | 信创系统 WebView 版本和发行版依赖可能不同。 | 在统信 UOS、银河麒麟和目标 CPU 上逐项验证。 |
| 字体渲染差异 | 中文字体、emoji 图标、代码字体可能显示不同。 | 使用系统字体栈并做截图验收。 |
| 高 DPI 缩放差异 | 不同系统缩放策略不同。 | 覆盖 2K/4K 与多缩放比例测试。 |
| 浏览器内核差异 | 信创浏览器可能与 Chrome/Edge 行为不同。 | 对文件能力、下载能力、localStorage 做专项测试。 |
