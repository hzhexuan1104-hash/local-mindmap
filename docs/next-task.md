# v1.3.0 发布后下一步任务建议

## 优先建议

1. Tauri 环境补齐与桌面端验证：
   - 安装 Rust / Cargo。
   - 在 Windows 上运行 `npm run tauri:dev` 和 `npm run tauri:build`。
   - 验证桌面窗口启动、加载当前 Vite 应用、保存/打开 `.lmind`、导入导出和图片导出。
   - 继续验证 macOS、Linux、统信 UOS、银河麒麟和国产 CPU 架构。
2. 节点拖拽体验补强：
   - 验证 1000 节点导图拖拽响应。
   - 优化拖拽过程中的性能和选中态。
   - 评估是否需要吸附、辅助线或简单避让。
   - 评估兄弟节点拖拽排序、批量层级调整和更智能的排版辅助。
3. 多选能力继续增强：
   - 增加批量移动、批量样式编辑。
   - 优化框选后的批量操作体验。
   - 评估多选复制 / 剪切 / 粘贴后的批量移动和批量样式编辑体验。
4. 自动化测试继续补充：
   - 增加 UI 组件测试或端到端测试。
   - 覆盖保存、打开、导入导出、插件启停、模板、主题、拖拽、多选、右键菜单等主流程。
5. UI 细节打磨：
   - 检查小屏和高 DPI 表现。
   - 优化顶部工具区密度。
   - 优化弹窗、右键菜单、备注面板和长文本溢出。

## 后续功能

- 兄弟节点拖拽排序和批量层级调整。
- 框选后的批量移动和批量样式编辑。
- 粘贴位置细节增强，例如吸附、辅助线或更智能的避让。
- 更完整的主题和线条样式配置。
- 插件签名、插件目录和沙箱执行能力。
- 桌面端真实安装包 Release。

## v1.3.0 发布前交互修复：框选与画布平移

本次只修复画布框选和平移的交互冲突，不新增复杂框选模式、不恢复兄弟节点拖拽排序、不修改 `.lmind` 结构。

- 已完成：画布空白区域普通左键拖拽用于平移画布。
- 已完成：Shift + 画布空白区域左键拖拽用于框选节点，结果替换当前选择。
- 已完成：框选命中改为节点中心点在框选矩形内才选中，减少边缘重叠导致的误选。
- 已完成：框选、节点拖拽和平移过程中阻止浏览器原生文字选区，输入框和备注编辑区仍可选择文本。
- 已完成：补充 `boxSelection.test.ts` 覆盖中心点命中、zoom / pan 坐标换算、Shift 框选和普通空白拖拽平移入口。
- 已完成：修复 root / 中心主题向左或向上拖动被布局边界卡住的问题，允许负坐标 position 保存和打开恢复。
- 已完成：修复非 Shift 框选后旧主选节点残留高亮的问题，`selectedNodeId` 与 `selectedNodeIds` 保持一致。
- 已完成：root 仍不能被拖成其他节点子节点，但 root position 自由拖动不受结构保护影响。
- 已完成：点击画布空白区域清空选择，不再默认选中中心主题，后续 Ctrl / Shift 多选不会自动带上 root。
- 已完成：框选视觉矩形和节点命中统一使用同一套 screen/world 坐标转换，修复 zoom / pan 后视觉与命中偏差。

# v1.4 后续任务建议

当前第一批已完成桌面 Native 插件目录和 manifest 管理底座。下一步建议：

1. 在本机执行 `rustup default stable` 或配置公司内网认可的 Rust toolchain 后，回归 `npm run tauri:dev` 和 `npm run tauri:build`。
2. 在 Windows 上手动验证桌面插件目录创建、manifest 安装、扫描、启用、禁用、卸载。
3. 在 macOS / Linux 上验证 Tauri app data dir 映射到预期插件目录。
4. 继续保持 v1.4 边界：不加载 DLL、不执行第三方代码、不做远程插件市场。
5. v1.5 再评估 Windows DLL 加载实验版，包括 ABI、可信来源、签名、崩溃隔离和权限模型。
