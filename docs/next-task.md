# 下一步任务建议

## 优先建议

1. 桌面端真实打包：
   - 创建 Tauri 或 Electron 最小配置。
   - 保持 Web 版和 GitHub Pages 不受影响。
   - 验证 Windows、macOS、Linux / 信创系统。
2. 自动化测试继续补充：
   - 增加 UI 组件测试或端到端测试。
   - 覆盖保存、打开、导入导出、插件启停、模板、主题等主流程。
3. UI 细节打磨：
   - 检查小屏和高 DPI 表现。
   - 优化顶部工具区密度。
   - 优化弹窗、面板和长文本溢出。
4. 性能优化：
   - 准备 1000 节点测试文件。
   - 测量打开、编辑、导出耗时。
   - 评估大导图渲染和 Markdown 预览性能。
5. 创建 GitHub Release：
   - 按 `docs/release-checklist.md` 完成发布检查。
   - 使用 `docs/release-notes-v1.0.0.md` 作为 Release 说明基础。
   - 记录版本号、已知风险和验证结果。
   - 等 GitHub Actions 通过后创建 Release。

## 后续功能

- 节点复制 / 剪切 / 粘贴。
- 拖拽节点调整层级与顺序。
- 框选多个节点和批量操作。
- 更完整的主题和线条样式配置。
- 插件签名、插件目录和沙箱执行能力。
