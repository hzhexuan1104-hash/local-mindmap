# v1.2.0 发布说明草稿

发布日期：待定

## 版本定位

v1.2 目标是在不破坏 v1.1.1 UI 信息架构和既有编辑能力的前提下，继续补齐基础思维导图编辑体验。第一批补齐节点复制 / 剪切 / 粘贴，第二批补齐框选基础版、Ctrl+A 全选和快捷键帮助，第三批补齐拖拽调整父子层级基础版。

本版本继续坚持：

- 纯本地运行。
- 不上传用户文件或节点数据。
- 不引入云端同步、用户登录或协同编辑。
- 不执行第三方插件 JavaScript。
- `.lmind` 数据结构不变。

## 新增功能

### 第一批：复制 / 剪切 / 粘贴

- 节点复制：
  - 支持复制当前主选中节点。
  - 支持复制多个选中节点。
  - 复制完整子树，包含节点文本、备注、children、nodeTypeId、collapsed 和可选 position。
  - root 节点可以复制，但粘贴后只作为普通子节点，不覆盖中心主题。
- 节点剪切：
  - 支持剪切当前主选中节点或多个选中节点。
  - root 节点不能剪切。
  - 多选中包含 root 时自动跳过 root。
  - 剪切采用内部剪贴板记录，粘贴成功后从原位置删除。
- 节点粘贴：
  - 支持 Ctrl+V 和右键菜单粘贴。
  - 默认粘贴到当前主选中节点下；无有效选中节点时粘贴到 root 下。
  - 粘贴时为所有节点重新生成 ID，避免重复 ID 和循环结构。
  - 粘贴后选中新粘贴的节点。
  - 目标节点有 position 时，新节点靠近目标节点右侧；目标节点无 position 时交给自动布局处理。
- 复制为同级节点：
  - 支持 Ctrl+D。
  - 非 root 节点复制到同一父节点下。
  - root 节点复制为 root 的普通子节点。
- 新增入口：
  - 顶部编辑菜单：复制节点、剪切节点、粘贴节点、复制为同级节点。
  - 节点右键菜单：复制节点、剪切节点、粘贴为子节点、复制为同级节点。
  - 画布右键菜单：粘贴到中心主题、清空内部剪贴板。
  - 快捷键：Ctrl+C、Ctrl+X、Ctrl+V、Ctrl+D。

### 第二批：框选与快捷键

- 框选节点基础版：
  - 在画布空白区域按下并拖动可以显示半透明框选矩形。
  - 框选覆盖到的节点会被选中。
  - 不按 Shift 时，框选结果替换当前选择。
  - 按住 Shift 框选时，命中节点追加到当前选择。
  - 框选命中支持当前画布缩放和位移。
  - 节点拖拽、右键菜单、工具栏和备注面板不触发框选。
- Ctrl+A 全选节点：
  - 非输入状态下选中当前导图所有节点。
  - 输入框、备注 textarea、select 和可编辑元素内不拦截浏览器默认 Ctrl+A。
  - 全选后可批量删除普通节点、批量切换节点类型和复制节点，root 删除保护继续生效。
- Esc 行为统一：
  - 优先关闭弹窗。
  - 其次关闭右键菜单。
  - 框选中按 Esc 取消框选。
  - 无弹窗、菜单和框选时清空当前选择到中心主题。
- 快捷键帮助完善：
  - 左侧工具栏和工具菜单均可打开快捷键帮助弹窗。
  - 弹窗列出 Ctrl+Z、Ctrl+Y、Ctrl+C、Ctrl+X、Ctrl+V、Ctrl+D、Ctrl+A、Ctrl+S、Ctrl+O、Delete、Backspace 和 Esc。
  - 支持 Esc 关闭和关闭按钮关闭。

### 第三批：拖拽调整父子层级

- 拖拽调整父子层级基础版：
  - 普通节点拖拽仍保留自由移动 `position` 的能力。
  - 拖拽节点到另一个有效节点附近时，目标节点显示可放置高亮。
  - 松开鼠标时，如果释放点命中有效目标，则被拖拽节点移动为目标节点的子节点。
  - root 中心主题不能被移动为其他节点的子节点。
  - 不允许移动到自己或自己的子孙节点下，避免产生循环结构。
  - 移动后节点 id 不变化，text、remark、nodeTypeId、collapsed、position 和 children 不丢失。
  - 移动操作进入撤销 / 重做历史。
- 树结构安全校验：
  - 新增父节点查找、子孙判断、移动为子节点和树完整性校验纯函数。
  - 校验 rootNode 始终存在、节点 ID 不重复、树结构不被循环或复用对象破坏。
- 本批仍是基础版：
  - 不包含兄弟节点拖拽排序。
  - 不包含复杂智能排版、鱼骨图、双向脑图或新布局。
  - 多选时只保证主拖拽节点的层级调整。

## 数据兼容

- 不修改 `.lmind` 文件结构。
- 内部剪贴板状态不保存到 `.lmind`。
- 框选状态和快捷键帮助弹窗状态不保存到 `.lmind`。
- 旧 `.lmind` 文件仍可正常打开。
- JSON 导入导出继续保留 `position`。
- 拖拽目标高亮状态不保存到 `.lmind`。
- 拖拽调整层级只修改既有 `children` 树关系，不新增文件结构字段。
- 粘贴节点会重新生成 ID，避免重复 ID。

## 测试建议

- 执行 `npm run test`。
- 执行 `npm run build`。
- 验证 Ctrl+C / Ctrl+X / Ctrl+V / Ctrl+D。
- 验证画布空白区域拖动框选、Shift 追加框选、Ctrl+A 全选和 Esc 关闭 / 清空选择。
- 验证快捷键帮助弹窗入口、内容、Esc 关闭和关闭按钮。
- 验证普通节点拖拽到目标节点附近时目标高亮，松开后成为目标子节点。
- 验证 root 不能被拖成其他节点子节点，节点不能拖到自己或子孙节点下。
- 验证拖拽调整层级后保存 `.lmind` 并重新打开，父子关系正确恢复。
- 验证顶部编辑菜单、节点右键菜单和画布右键菜单入口。
- 验证复制粘贴后子树完整，备注、节点类型、折叠状态和 position 行为符合预期。
- 验证 root 不能剪切，root 复制后只能作为普通子节点。
- 验证粘贴、剪切粘贴和复制为同级节点可撤销 / 重做。
- 保存 `.lmind` 后重新打开，确认粘贴后的结构正常。
- 回归 v1.1.1 的顶部菜单、左侧抽屉、备注折叠、专注模式、拖拽、多选和右键菜单。

## 已知限制

- 第二批只包含框选基础版，不包含框选后的批量移动或批量样式编辑增强。
- 第三批只包含拖拽调整父子层级基础版，不包含兄弟节点拖拽排序、批量层级调整或复杂智能排版。
- 本版本不包含 PDF 导出、云同步、协同编辑或插件市场。
- 自动化测试覆盖剪贴板、框选、快捷键和树操作纯函数；完整浏览器交互仍需要按验收清单手动回归。
